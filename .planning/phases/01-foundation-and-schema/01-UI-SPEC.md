## Interaction Contract: Field Visibility (FOUND-02)

This is the only interaction contract in Phase 1. It governs how fields visually appear and
disappear in the SurveyBuilder preview when `isFieldVisible()` from `lib/survey-logic.ts` is
evaluated.

### Visible State

A field whose `showIf` condition evaluates to `true` (or has no `showIf`) renders at full opacity
with normal layout flow. No visual indicator is shown — the field appears as it always has.

### Hidden State (builder preview only)

In the SurveyBuilder preview, a field whose `showIf` condition evaluates to `false` is not
removed from the DOM. It renders with:

- `opacity: 0.35` — visually dimmed to indicate conditional invisibility
- `pointer-events: none` — non-interactive
- A text label: "Hidden by condition" displayed in `--muted-foreground` (#78716c) at 12px/400
  immediately below the field label, using `font-style: italic`

Rationale: Removing the field entirely in builder preview would confuse the builder user who needs
to see that the field exists and is conditionally hidden. Dimming is the established pattern for
conditional state in visual editors.

### Hidden State (public form)

On the public survey form (`app/s/[slug]/page.tsx`), a field whose `showIf` condition evaluates to
`false` is not rendered at all (current behavior preserved). No DOM node, no opacity trick. This is
the existing inline behavior being replaced with the shared evaluator — visual output is unchanged.

### Transition

Field visibility changes in the builder preview animate with `transition: opacity 150ms ease-out`.
No transition on the public form (fields are server-rendered per page step).
