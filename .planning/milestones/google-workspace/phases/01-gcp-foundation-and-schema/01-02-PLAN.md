| Threat | Severity | Mitigation |

    ```sql
      "google_account_id" varchar(64) NOT NULL,
      "access_token" text NOT NULL,
      "refresh_token" text NOT NULL,

      "calendar_sync_token" text,
      "contacts_sync_token" text,
      "last_sync_at" timestamp,
      "revoked_at" timestamp,
      "created_at" timestamp DEFAULT now() NOT NULL,
      "updated_at" timestamp DEFAULT now() NOT NULL,
      CONSTRAINT "google_workspace_user_connections_client_user_unique" UNIQUE ("client_id", "user_id")
    );

    CREATE INDEX "google_workspace_user_connections_user_idx"
      ON "google_workspace_user_connections" ("user_id");

    -- ─── crm_activities: provenance column ────────────────────────────────────

    ALTER TABLE "crm_activities"
      ADD COLUMN "via_user_id" integer REFERENCES "users"("id") ON DELETE SET NULL;

    CREATE INDEX "crm_activities_via_user_idx"
      ON "crm_activities" ("via_user_id");
    ```

    CRITICAL design choices baked into the SQL:
    - **Defaults differ between client and user connections**: client connections default to `aggressiveness=moderate, storeBodies=true` (org data, agency wants the long tail). User connections default to `aggressiveness=passive, storeBodies=false` (personal inbox, conservative privacy). UI can override on first connect.
    - **`google_account_email` is varchar(320)**: RFC 5321 max email length is 254 chars but RFC 5322 allows 320. Match existing schema patterns or pick larger of the two.
    - **`google_account_id` is varchar(64)**: Google's `sub` claim is a numeric string up to 21 digits but reserve more.
    - **Sync cursor columns are nullable**: a fresh connection has none until first sync.
    - **`revoked_at` is nullable**: NULL = active, non-NULL = soft-revoked.
    - **No `unique(client_id)` on user_connections** — that's the per-client *client* connection. User connections are unique on (client_id, user_id).
    - **ON DELETE cascade for client_id and user_id** — when a client or user is deleted, their connections go with them. Soft-delete of users (if any) is unaffected.
