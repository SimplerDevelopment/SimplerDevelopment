# Feature Inventory — Product Domains

Generated from the Domain Maps in `vault/03 - Domains/`. Each section derives from the canonical map; gaps and stubs are flagged explicitly.

## 1. Sites, Hosting & Publishing

**Purpose:** Manages per-tenant websites from infrastructure provisioning through public rendering. Three sub-concerns: Sites (clientWebsites, domain routing, custom CSS/JS, nav, branding), Hosting (Railway-backed managed apps, Vercel-backed CMS), and Publishing (multi-channel content staging pipeline using a hidden kanban project).

**Target user:** Agency admin, client (portal)

**Key routes:**
- `app/portal/websites/` — site list, new site, per-site settings
- `app/portal/websites/[siteId]/` — branding, navigation, code, content-types, taxonomy, posts, calendar
- `app/portal/hosting/`, `app/portal/hosting/[id]/`
- `app/portal/publishing/board/`, `/calendar/`, `/campaigns/`, `/permissions/`, `/tags/`
- `app/portal/snapshots/`
- `app/sites/[domain]/[[...slug]]/` — public site render (SSR)
- `app/admin/portal-websites/`, `app/admin/portal-hosting/`

**Primary screens:** Site list, site settings, branding editor, navigation editor, custom code editor, per-site content calendar, hosting list/detail, publishing kanban board, publishing calendar, publishing campaigns, publishing permissions, snapshot export/import.

**MCP tools:** Yes — `sites_list`, `sites_update`, `sites_get/update/publish_custom_code`, `nav_list/create/update/delete/publish/publish_all`, `website_domains_list/add/remove`, `website_env_vars_list/set/delete`, `hosting_list`, `hosting_get`.

**Flags:** Social and webhook publishing channels are not built — only the email channel exists under `lib/publishing/channels/`. `site_snapshots.isPublic` marketplace feature has no logic. Full DB-lookup middleware host-header gate deferred to Wave 3.

## 2. CMS, Posts & Blocks

**Purpose:** Content management layer for all tenant websites. Posts store a JSON block tree (`{ blocks: Block[], version: '1.0' }`). Covers 48+ registered block types, post types, custom fields, taxonomies, block templates, media library, and block-based email templates.

**Target user:** Agency admin, client

**Key routes:**
- `app/portal/websites/[siteId]/posts/` — post list, new post, post detail, visual editor (`/edit/`)
- `app/portal/media/` — global media library
- `app/portal/websites/[siteId]/media/` — per-site media
- `app/sites/[domain]/[[...slug]]/` — public render
- `app/api/portal/cms/websites/[siteId]/` — posts, categories, tags, block-templates, content-types, media, taxonomies, email-templates, tracking

**Primary screens:** Post list, new post wizard, post detail/settings, visual block editor, block template picker, media library, content type manager, taxonomy manager, tracking settings.

**MCP tools:** Yes — 42 tools covering posts, sites, taxonomies, block templates, navigation, media, domains, env vars. Registered via `registerCmsTools()` in `lib/mcp/tools/cms.ts`.

**Flags:** `roi-calculator` block settings panel is partially wired (16 inputs not yet connected, title/description only — baselined as known debt). Block type lockstep across 9 locations remains a manual multi-step process.

## 3. Visual Editor

**Purpose:** Block-based WYSIWYG page builder embedded in the portal. The portal shell owns block state; a sandboxed iframe loads the live public-site renderer. Communication is a typed postMessage protocol. Supports drag-and-drop, breakpoint previews, inline editing, undo/redo, multi-user collaboration presence, and contextual AI restyle.

**Target user:** Agency admin, client

**Key routes:**
- `app/portal/websites/[siteId]/posts/[postId]/edit/` — full editor

**Primary screens:** Block picker (left panel), layers panel, settings sidebar (right panel), element style editor, collaboration presence bar, context menu.

**MCP tools:** No dedicated tools. Block content is written through CMS tools (`posts_create`, `posts_update`). The `blocks://schema` MCP resource provides the block JSON schema to AI clients.

**Flags:** No known stubs. Full E2E spec (`visual-editor-blocks.spec.ts` — 1871 lines). Voice assistant widget is built but not mounted in the portal layout.
