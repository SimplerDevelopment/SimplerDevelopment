## What's Next: Automation Engine

The user wants a dual-layer automation system:

### Layer 1: Per-Product Settings Screens
Dedicated UI pages within each product for standard automations:
- **Booking**: toggle confirmation emails, reminder timing/channel, follow-up delay, no-show actions
- **Email**: autoresponder config, drip sequence timing, engagement branching
- **Websites**: form action routing, cart recovery settings, post-purchase flows
- **Projects**: notification rules (status change, due date), auto-assign rules
- **AI Chat**: escalation rules, after-hours behavior, confidence thresholds
- **CRM**: pipeline stage actions, contact scoring, deal-won onboarding

### Layer 2: NLP-Powered Custom Rule Creator
Users describe automations in plain English, AI parses into structured rules:
- "When someone books an appointment, send them a confirmation email and create a task for my team"
- Parses to: trigger=booking.confirmed, actions=[email.send, project.createTask]
- Uses same tool-calling system as the AI chat bot (30+ portal tools already exist)
- No n8n dependency, no visual flowchart — just a text input + AI

### Architecture Needed
1. **`automation_rules` table** — stores NLP description + parsed trigger/condition/action JSON + product scope + enabled flag
2. **Event bus** — API routes emit events (booking.created, form.submitted, email.clicked, etc.)
3. **AI parser** — takes natural language -> structured trigger/condition/action config (Claude tool call)
4. **Executor** — matches events to rules, runs actions using existing service functions
5. **Per-product settings pages** — structured UI that writes to same automation_rules table
6. **Portal automation management page** — list all rules, create via NLP, enable/disable

### Cross-Product Automations (the differentiator)
- Form -> CRM contact -> Email sequence -> Booking
- Booking confirmed -> Prep task + CRM update + Email
- Chatbot qualifies lead -> Book appointment -> Create deal
- Deal won -> Create project from template -> Welcome email -> Schedule kickoff
- Purchase -> Thank you email -> Review request (14 days) -> Referral program

### Key Design Decisions
- No dependency on n8n — pure internal execution using existing service functions
- NLP-first for custom automations, structured UI for common per-product settings
- Both layers write to the same `automation_rules` table
- Event-driven: API routes emit events, engine matches and executes
- AI token usage for NLP parsing counts against client's credit budget
